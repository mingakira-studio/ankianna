## Research 发现

### 现有代码状态
1. **DesignTokens.swift** — 已建立基础（Colors/Font/Spacing/Radius/Shadow/Animation），但：
   - Shadow 仅 radius=2，无多层深度
   - Radius 最大 xl=24，Claymorphism 需 40+
   - Colors 不区分 Light/Dark（canvas=.white 暗色不可用）
   - Font 硬编码 CGFloat，不支持 Dynamic Type

2. **CelebrationEffects.swift** — ConfettiView 用 12 个 SF Symbols + 简单动画（offset/rotation/opacity），非 SpriteKit

3. **暗色模式** — 完全未适配，无任何 colorScheme 检查

4. **Dynamic Type** — DesignTokens.Font 使用 .system(size:) 硬编码，不跟随系统设置

5. **Onboarding** — 不存在，应用直接进入主视图

## 风险与边界
- SpriteKit 粒子在 iPad 上性能需测试
- Dynamic Type 字体溢出布局需逐视图验证
- Claymorphism 多层阴影在暗色模式下可能看不清
- 工作量大（25+ 文件），建议分子任务逐步推进

## 技术方案
- 目标: 完成 P4 进阶优化的 5 个子项，提升视觉品质和可访问性
- 方案: SpriteKit粒子(SKEmitterNode) → Dynamic Type(@ScaledMetric/.relativeTo) → Claymorphism(多层阴影+ViewModifier) → 暗色模式(Color light/dark) → Onboarding(AppStorage+3页引导)
- 边界: 不重构业务逻辑/ViewModel，不新增功能，Claymorphism 只应用到卡片/按钮级组件
- 交付层级: 视觉升级 + 可访问性增强
- 验收标准: iPad 上切换暗色模式/Dynamic Type 均正常显示，庆祝特效流畅，首次启动有引导
