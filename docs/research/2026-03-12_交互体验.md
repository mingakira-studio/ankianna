# Research: UI优化 P2 — 交互体验
日期: 2026-03-12
深度: 轻量

## 任务上下文
审查报告 P2 章节包含 5 个优化项（N2/M2/A4/N1/I2），均为视图层交互改进，无业务逻辑变更。

## Research 发现
- N2: 4个游戏模式结束画面仅有"再来一次"，无返回首页按钮；用 @Environment(\.dismiss) 即可
- M2: 游戏状态切换（start→gameplay→gameOver）无过渡动画；用 .transition() + withAnimation
- A4: CardLibraryView 正确率和 StreakCalendarView 打卡仅用颜色区分；需添加图标辅助
- N1: 学习 tab 无 badge 提示待复习数；需 Query CharacterStats + .badge() modifier
- I2: ResultFeedbackView 答对只显示"太棒了！"不显示对应字；correctAnswer 已传入，补显示即可

## 关联文件
- ContentView.swift — tab badge
- TimeAttackView.swift / SurvivalView.swift / LevelsView.swift / MatchView.swift — 返回首页按钮 + 过渡动画
- ResultFeedbackView.swift — 答对显示字
- CardLibraryView.swift — 颜色辅助指示
- StreakCalendarView.swift — 打卡颜色辅助

## 风险与边界
- 过渡动画不宜过重，影响游戏流畅度
- Tab badge 查询需高效，避免影响启动性能
- 纯 UI 变更，不涉及数据模型或业务逻辑

## 技术方案
- 目标: 完成审查报告 P2 的 5 项交互体验优化（N2/M2/A4/N1/I2）
- 方案: N2 用 dismiss() 返回首页；I2 在 ResultFeedbackView 补显示字；N1 查询 CharacterStats due 数 + .badge()；A4 正确率旁加图标；M2 加 .transition + withAnimation
- 边界: 不改数据模型、不改业务逻辑、不做 Claymorphism
- 交付层级: 视图层纯 UI 优化
- 验收标准: 游戏结束能返回首页、答对看到字、学习 tab 有数字 badge、正确率有图标辅助、状态切换有动画
