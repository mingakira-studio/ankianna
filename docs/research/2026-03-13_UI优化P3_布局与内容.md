## Research 发现

### L1: 游戏模式未适配 iPad 横屏 (HIGH)
- TimeAttackView, SurvivalView, LevelsView, MatchView 的 gameplayView 全用 VStack
- 横屏时竖向布局导致巨大留白
- 需检测 horizontalSizeClass 或 GeometryReader，横屏时改 HStack 左右分栏
- MatchView 额外问题: LazyVGrid 固定 4 列，横屏可增加列数

### L2: GameModeCard 固定高度 (HIGH)
- GameModeSelectionView 第 99 行 frame(height: 160) 硬编码
- iPad 大屏上卡片偏小
- 改为 adaptive（GeometryReader 或 minHeight + flexible）

### I3: Session 完成页无分享 (MEDIUM)
- LearningView 第 427-461 行 sessionCompleteView
- 仅显示统计，无分享按钮
- 需添加 ShareLink 或 UIActivityViewController
- 文本分享优先（截图生成复杂度高，可后续）

### M3: MascotView bounce 双重赋值 (LOW)
- MascotView 第 48-50 行 bouncing=true 赋值两次
- 删除 withAnimation 外的第一行即可

## 关联文件
- 修改: GameModeSelectionView.swift, TimeAttackView.swift, SurvivalView.swift, LevelsView.swift, MatchView.swift, MascotView.swift, LearningView.swift
- 可能新增: 无（分享用 ShareLink 内联即可）

## 风险与边界
- horizontalSizeClass 在不同 iPad 尺寸上行为差异 → 优先用 GeometryReader 检测宽度
- WritingCanvas 在 HStack 中需保证 minHeight
- 仅适配 iPad 横屏，iPhone 保持竖屏
- Session 分享仅文本，不做截图
- 回归: 161 unit tests + 6 screenshot tests 必须通过

## 技术方案
- 目标: iPad 横屏合理布局 + GameModeCard 自适应高度 + MascotView bug 修复 + Session 分享
- 方案: horizontalSizeClass + GeometryReader 检测横屏，4 游戏模式 HStack 分栏；GameModeCard 去硬编码改 flexible；ShareLink 文本分享
- 边界: 不做 iPhone 横屏、不做截图分享、不改 Grid 列数、不引入新依赖
- 交付层级: 视图层 SwiftUI 修改
- 验收标准: iPad 横屏左右分栏无大面积留白；GameModeCard 高度自适应；分享按钮弹出系统面板
