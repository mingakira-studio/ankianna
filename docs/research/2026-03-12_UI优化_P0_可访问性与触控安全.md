## Research 发现

### 1. 触控目标问题
- CardLibraryView:92-98 — trash 按钮 `.font(.system(size: 15))` + `.buttonStyle(.plain)`, 无 frame/padding 扩大触控区域
- StreakCalendarView:31-33 — 日历圆点 `.frame(width: 24, height: 24)`, 低于 44pt 最低要求
- 其他按钮（speaker、game mode cards、list rows）触控面积足够

### 2. 可访问性 (Accessibility)
- **零 accessibilityLabel/Hint** — 全部视图仅有 accessibilityIdentifier（测试用），无语义标签
- 关键 icon-only 按钮缺标签: CardPromptView speaker, CardLibraryView trash
- 颜色作为唯一信息载体: 正确率绿/橙/红 (CardLibraryView), 日历绿/灰 (StreakCalendarView)

### 3. Reduced Motion 支持
- **零 reducedMotion 检查** — 所有动画无条件运行
- 受影响文件:
  - CelebrationEffects.swift: ConfettiView(offset/rotation/scale), ComboFireView(永久循环), EncouragementView(scale循环)
  - MascotView.swift: bounce spring 动画
  - ResultFeedbackView.swift: checkmark/points/combo 缩放动画
  - LearningView.swift: combo counter transition, practice feedback animation

### 4. 按压反馈 (Press Feedback)
- GameModeSelectionView:50 — `.buttonStyle(.plain)` 无按压视觉反馈
- CardLibraryView:99 — trash `.buttonStyle(.plain)` 无反馈
- 其他按钮用 `.borderedProminent` 有系统默认反馈

### 5. Haptic 反馈
- **完全缺失** — 无任何 UIFeedbackGenerator 或 .sensoryFeedback 使用
- 关键触点: 答对/答错、combo、游戏开始/结束、tile 匹配

### 6. exitFeedback 不可跳过
- LearningView:417 — `DispatchQueue.main.asyncAfter(deadline: .now() + 1.5)` 强制等待
- 无 tap gesture 允许提前跳过

## 风险与边界
- Haptic 在 Simulator 不触发，需真机验证体感
- reducedMotion 需开启系统无障碍设置测试
- 颜色辅助指示（A4）属于 P2 任务范围，P0 只加 accessibilityLabel

## 技术方案
- 目标: 解决审查报告 P0 级别的 5 类问题（A2-A3, T1-T5），提升可访问性和触控安全
- 方案: 触控目标≥44pt、icon-only 按钮加 accessibilityLabel、reducedMotion 检查、GameModeCard 按压反馈、HapticService 关键触点触觉反馈、exitFeedback 可跳过
- 边界: 不改颜色辅助(P2)、不改 Dynamic Type/设计 token/字体/emoji 图标(P1)
- 交付层级: 代码修改，影响 10+ 视图文件 + 新增 HapticService
- 验收标准: 所有按钮触控≥44pt，icon-only 按钮有语义标签，reducedMotion 下无持续动画，GameModeCard 有按压反馈，关键操作有 Haptic，exitFeedback 可点击跳过
