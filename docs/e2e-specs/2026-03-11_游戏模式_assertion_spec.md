# 断言 Spec: 游戏模式

## 变更描述
学习 tab 从单一 LearningView 升级为 5 种游戏模式选择入口（快速学习/限时挑战/生存模式/闯关模式/连连看）

## 用户可见影响
打开学习 tab 看到模式选择网格而非直接进入学习；可选择不同模式体验不同玩法

## Hard Assertions（必须自动化）

| # | 断言 | 维度 | 验证方法 |
|---|------|------|---------|
| H1 | 学习 tab 显示"快速学习"文本 | 存在性 | `app.staticTexts["快速学习"].exists` |
| H2 | 学习 tab 显示"限时挑战"文本 | 存在性 | `app.staticTexts["限时挑战"].exists` |
| H3 | 学习 tab 显示"生存模式"文本 | 存在性 | `app.staticTexts["生存模式"].exists` |
| H4 | 学习 tab 显示"闯关模式"文本 | 存在性 | `app.staticTexts["闯关模式"].exists` |
| H5 | 学习 tab 显示"连连看"文本 | 存在性 | `app.staticTexts["连连看"].exists` |
| H6 | 点击"快速学习"进入学习界面 | 可达性 | tap后验证学习界面元素出现 |
| H7 | 点击"限时挑战"显示时长选择 | 可达性 | tap后`app.buttons["60 秒"].exists` |
| H8 | 限时挑战选择时长后显示倒计时 | 响应性 | 选60秒后验证倒计时元素存在 |
| H9 | 点击"生存模式"显示命数 | 可达性+存在性 | tap后验证命数/心形图标存在 |
| H10 | 点击"闯关模式"显示关卡网格 | 可达性+存在性 | tap后验证关卡元素存在 |
| H11 | 闯关模式第一关默认解锁 | 正确性 | 第一关按钮可点击 |
| H12 | 点击"连连看"显示配对网格 | 可达性+存在性 | tap后验证 tiles 网格存在 |
| H13 | 现有"快速学习"流程不回归 | 一致性 | 通过模式选择进入后，学习流程正常 |

## Manual-Only

| # | 断言 | 原因 | 前置验证结果 |
|---|------|------|-------------|
| M1 | 模式卡片视觉美观、布局均匀 | 视觉设计主观判断 | 截图可辅助但无法判定"美观" |
| M2 | 各模式动画流畅 | 动画性能需真人感知 | 帧率可量化但主观感受不同 |
