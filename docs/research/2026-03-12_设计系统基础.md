## Research 发现

### 现状
1. **无设计 token 系统** — 颜色/字号/间距/圆角全部硬编码内联，分散在 28 个视图文件中
2. **字体** — 全部使用默认 San Francisco，无 SF Rounded（`.design(.rounded)`）使用
3. **吉祥物** — MascotView.swift 纯 emoji（🦕🤔🥳💪🎉），font size 50pt + 弹跳动画 + 语音泡泡
4. **游戏模式视觉不一致**:
   - 圆角: 10/12/16 三种混用
   - 间距: 4/8/12/16/20/24 无规律
   - 按钮样式: `.borderedProminent`/`.bordered`/`.plain` 混用
   - 颜色: GameMode enum 有 5 色，但其余颜色散落各处

### 关键数值统计
| 类别 | 当前值 | 建议统一 |
|------|--------|----------|
| 圆角 | 10, 12, 16 | sm=8, md=12, lg=16, xl=24 |
| 间距 | 4,6,8,12,16,20,24 | 4pt scale: 4/8/12/16/24/32 |
| 字号 | 硬编码 12-80pt | 语义 token: caption/body/title/display |
| 颜色 | .orange/.red/.pink/.blue/.green + systemGray6 | 语义 token: primary/accent/success/warning/error/surface |

## 风险与边界
- **emoji 替换**: Lottie 需外部美术资源，改用 SwiftUI 几何形状绘制简单恐龙角色
- **token 迁移范围大**: 28 个视图文件，需分批替换避免引入回归
- **SF Rounded 全局应用**: 可通过 `.environment(\.font)` 或逐文件替换
- **不做**: Claymorphism 风格（P4）、Dynamic Type（P4）、SpriteKit 粒子（P4）

## 技术方案
- 目标: 建立统一设计系统，消除硬编码值，所有视图共享一致视觉语言
- 方案: DesignTokens.swift 定义 token → SF Rounded 字体工厂 → SwiftUI 绘制恐龙替换 emoji → 逐批迁移视图
- 边界: 不做 Claymorphism/Dynamic Type/SpriteKit 粒子（P4），纯视觉层改动不改业务逻辑
- 交付层级: 设计系统基础 + 全量视图迁移（16 个文件）
- 验收标准: 所有页面 SF Rounded 字体，颜色/间距/圆角用 DesignTokens，吉祥物为 SwiftUI 绘制恐龙
